# 棚卸し(Sync)手順

この手順は、ユーザーが明示的に「棚卸しして」「sync して」と依頼したときにのみ実行する。
自発的に実行しないこと。

棚卸しには2段階ある。まずどちらを行うかユーザーに確認する。

- **ライト棚卸し**: Skills / Agents の差分確認のみ。数分で完了。
- **フル棚卸し**: 上記に加えて Memory のスナップショット取得と差分分析。

---

## 事前準備: claude.ai 側からの情報取得

この作業は Claude Code だけでは完結しない。ユーザーに claude.ai のチャットで
以下を実行してもらい、その出力を貼ってもらう。

**Skills / Agents 一覧を取得するプロンプト例:**

> 今あなたが使えるスキルの一覧を、名前と description の1行要約でリストアップして。
> ユーザースキル・公式スキル・example スキルを区別して。

**Memory を取得するプロンプト例（フル棚卸しのときのみ）:**

> これまでの会話から覚えていることを、カテゴリ別に箇条書きで整理して出力して。

出力が貼られたら、以下の手順に進む。

---

## Step 1: Skills の差分

1. `.claude/skills/` のフォルダ名一覧を取得する。
2. 貼られた claude.ai 側の一覧と突き合わせる。
3. 差分を次の3分類で報告する。

| 分類 | 意味 | 既定の扱い |
|---|---|---|
| 追加候補 | claude.ai にあるが playground にない | ユーザー承認後に追加 |
| 削除候補 | playground にあるが claude.ai で使われていない | 削除は必ずユーザー承認後 |
| 更新候補 | 両方にあるが内容が変わっていそう | 差分を見せて判断を仰ぐ |

削除は情報が失われる操作なので、判断に迷ったら残す方を選ぶ。

## Step 2: Skills の取り込み

追加・更新が承認されたスキルについて、SKILL.md の内容をユーザーから受け取り
（または claude.ai 側で書き出してもらい）、`.claude/skills/<name>/SKILL.md` として
配置する。

配置前に `skill-optimizer` で description の発火条件をチェックすると精度が上がる。
特に、既存スキルと description が競合していないかを見る。

## Step 3: Agents の差分

`.claude/agents/` に対して Step 1〜2 と同じことを行う。

## Step 4: Memory のスナップショット（フル棚卸しのみ）

1. ユーザーから貼られた記憶の要約を、そのまま
   `memory/snapshots/YYYY-MM-DD.md` として保存する。日付は取得日。
2. `memory/snapshots/` 内の直前のファイルと比較し、以下を探す。
   - 新しく登場した作業パターン・繰り返し
   - 前回から消えた（もう関心がなさそうな）トピック
   - 前提条件の変化（使用ツール、環境、役割の変更など）
3. 繰り返し登場するパターンが見つかったら `skill-harvester` に渡す。
   依頼文の例:
   > `memory/snapshots/` の直近2ファイルの差分から、Skill 化できそうな
   > 手順・パターンを探して提案して。

## Step 5: builtins-reconstructed / skill-library の更新

claude.ai や Claude Code に新機能が増えていた場合、該当するベストエフォート版
ドキュメントを追記・更新する。既存の `README.md` の表形式に合わせること。

新機能の有無が不明な場合は、以下を web fetch して確認できる。

- Claude Code ドキュメント索引: <https://code.claude.com/docs/en/claude_code_docs_map.md>
- 週次の更新履歴: 同索引内の "What's New" セクション

## Step 6: 記録

`memory/CHANGELOG.md` に1行追記する。フォーマット:

```
- YYYY-MM-DD: <ライト|フル> 棚卸し。skill +N/-M、agent +N、memory の主な変化を1文で。
```

## Step 7: コミット

ここまでの変更をまとめて1コミットにする。コミットメッセージ例:

```
sync: 2026-07-19 フル棚卸し (skill +3, memory snapshot 追加)
```

棚卸しは `CLAUDE.md` に定める「更新してよい例外ケース」に該当するため、
このコミットと push は実行してよい。ただし削除を含む場合のみ、
push 前にもう一度ユーザーに確認する。

---

## 棚卸しの頻度の目安

決まった周期は設けない。以下のタイミングが自然。

- claude.ai 側で新しいスキルを3つ以上作ったとき
- 別リポジトリでの作業中に「playground のスキルが古い」と感じたとき
- Claude Code / claude.ai に大きなアップデートがあったとき

`/loop` で定期的に「差分があるか確認するだけ」を回すことも可能だが、
claude.ai 側の情報取得に人手が要るため、実用上は手動起動が現実的。
