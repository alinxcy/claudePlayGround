# memory/

claude.ai 側の Memory（記憶）と Skills の状態を、このリポジトリにスナップショットとして
残しておくためのディレクトリです。

## なぜ半自動なのか

claude.ai の Memory は API 経由で機械的に取得できません。そのため完全自動同期はできず、
ここでの「同期」は **半自動**です。ユーザーが claude.ai 側で内容を書き出し、
Claude Code がそれをファイル化して差分を見る、という流れになります。

## ファイル構成

| パス | 内容 |
|---|---|
| `SYNC.md` | 棚卸し(sync)の実施手順。ユーザーが「棚卸しして」と言ったときに読む |
| `snapshots/` | 記憶とスキル一覧のスナップショット（`YYYY-MM-DD.md`） |
| `CHANGELOG.md` | 棚卸しの実施履歴（1回1行） |

## 基本の流れ

1. claude.ai のチャットで Claude に記憶とスキル一覧を書き出させる
2. その出力を `memory/snapshots/YYYY-MM-DD.md` として保存する
3. 前回のスナップショットと差分を取る
4. 差分から Skill 化すべきパターンがあれば `skill-harvester` に渡す
5. `CHANGELOG.md` に1行追記する

詳細は `SYNC.md` を参照してください。

## 注意

Memory には個人情報や機微な内容が含まれる可能性があります。このリポジトリを Public に
する場合は、`snapshots/` の内容を都度確認してから push してください。
機微な情報を含むスナップショットは `.gitignore` するか、要約レベルに留めるのが安全です。
